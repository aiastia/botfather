# 🧠 项目名称（暂定）

**TG Bot 托管 & 私聊助手平台**

基于 Telegram + aiogram

---

# 一、项目目标（Why）

构建一个系统，让用户：

* 提交自己的 bot（通过 BotFather 创建）
* 在 TG 内完成配置
* 拥有一个“私聊助手 bot”
* 支持插件扩展能力（AI / 自动化 / 记忆等）

---

👉 本质定位：

> 多租户 Bot 托管 + 插件化 AI 助手平台

---

# 二、核心功能需求（MVP）

---

## 2.1 主 Bot（控制入口）

用户与“主 bot”交互完成所有操作：

### 指令设计

```
/start         初始化
/add_bot       添加 bot
/my_bots       查看我的 bot
/delete_bot    删除 bot
/config        配置 bot
/help
```

---

## 2.2 Bot 接入流程

### 流程图

```
用户 → /add_bot
     → 输入 token
     → 系统校验 token
     → 创建 bot 实例
     → 设置 webhook
     → 完成接入
```

---

### Token 校验（必须）

调用 Telegram API：

```
getMe
```

校验：

* token 是否合法
* 是否为 bot
* 获取 bot_id / username

---

## 2.3 私聊助手功能（核心）

### 行为定义

* 仅允许 owner 使用
* 用户给 bot 发消息
* bot 返回响应

---

### 安全限制（必须）

```python
if message.from_user.id != owner_id:
    ignore
```

---

## 2.4 Bot 生命周期管理

支持：

* 添加 bot
* 启动 bot
* 停止 bot
* 删除 bot

# 三、插件系统设计（核心模块）

---

## 3.1 插件目标

实现：

* 功能解耦
* 可扩展
* 每个 bot 独立配置

---

## 3.2 插件接口定义

```python
class BasePlugin:
    name = "base"
    priority = 100

    async def on_message(self, bot, message, context):
        return PluginResult()
```

---

## 3.3 插件执行机制（核心）

```python
for plugin in plugins:
    result = await plugin.on_message(bot, message, context)

    if result.stop:
        break
```

---

## 3.4 插件返回结构

```python
class PluginResult:
    def __init__(self, stop=False):
        self.stop = stop
```

---

## 3.5 插件分类（建议）

---

### 1️⃣ 权限类

* Owner 校验
* 黑名单

---

### 2️⃣ 输入处理类

* 命令解析
* 关键词匹配

---

### 3️⃣ 逻辑处理类

* AI 对话（接 OpenAI）
* 数据处理
* 记忆系统

---

### 4️⃣ 输出类

* 回复用户
* 格式化消息
